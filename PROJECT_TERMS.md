# 项目使用协议 / Project Usage Terms

本文件用于说明本仓库在 Apache-2.0 许可基础上的协作与交付约定。

## 1. 许可关系 / License Relationship

1. 本仓库保留上游代码既有的 `Apache-2.0` 许可要求。
2. 若本文件与 `LICENSE` 内容冲突，以 `LICENSE`（Apache-2.0）为准。
3. 本文件补充的是协作规范、交付说明与使用建议，不改变上游开源许可条款。

## 2. 商业使用说明 / Commercial Usage

1. 允许将本仓库用于企业内部研发、PoC 与生产改造。
2. 商业部署前建议完成安全审计、压力测试和合规评估。
3. 由于部署环境差异导致的数据风险、停机风险由部署方自行承担。

## 3. 安全与密钥 / Security and Secrets

1. 禁止将真实 API Key、数据库密码提交到公开仓库。
2. 建议通过 `.env`、CI Secrets 或密钥管理服务注入敏感配置。
3. 建议定期轮换模型服务、数据库和缓存服务密钥。

## 4. 贡献规范 / Contribution Rules

1. 新增功能需包含最小可运行示例或测试用例。
2. 破坏性变更需在 PR 描述中明确迁移步骤。
3. 文档更新需与代码变更同步提交，避免文档漂移。

## 5. 免责声明 / Disclaimer

1. 项目按“现状”提供，不承诺适配所有环境。
2. 使用者应根据业务场景自行评估可用性、性能与合规性。
3. 因误配置、越权访问、第三方服务波动造成的损失，维护者不承担连带责任。
